# UI/UX Design System

Garuda should feel like a focused professional research workspace for venture analysis.

## Product Feel

The interface should be quiet, dense, structured, and optimized for repeated research work. It should prioritize scanning, comparison, filtering, writing, and decision support.

## Core Views

- Dashboard
- Startup list
- Startup profile
- Founder research
- Funding tracker
- Signal tracker
- Thesis workspace
- Scorecard
- Portfolio/watchlist

## Navigation

Use a persistent application shell with clear access to dashboard, startups, portfolio, signals, and research documents.

## Tables

Use TanStack Table for high-density startup, funding, signal, and portfolio views.

Tables should support:

- Sorting
- Filtering
- Status indicators
- Score columns
- Recent activity
- Quick navigation to startup profiles

## Charts

Use Apache ECharts for:

- Score distribution
- Signal volume over time
- Portfolio status mix
- Sector/category breakdown
- Funding activity summaries

## Forms

Forms should support fast manual entry. Use clear field labels, validation messages, and structured sections.

## Scorecards

Scorecards should show 1-5 scoring dimensions, notes per dimension, and an overall calculated score.

## Thesis Workspace

The thesis workspace should support long-form writing, structured prompts, and decision memo sections such as bull case, bear case, key questions, risks, and next action.

## Visual Tone

Use a restrained, work-focused visual style. Avoid marketing-style hero sections, decorative layouts, and unnecessary visual noise.
